## 📌 프로젝트 개요
### [모두의 플리 (MOPL)](https://woongsanam.kro.kr)

대규모 트래픽이 예상되는 글로벌 컨텐츠 평점 및 큐레이션 플랫폼입니다.  
영화, 드라마, 스포츠 등 다양한 콘텐츠를 큐레이팅하고 공유하며, 실시간 같이 보기 기능까지 제공하는 소셜 서비스입니다.  
사용자들은 자신만의 플레이리스트를 만들고, 다른 사용자와 소통하며 콘텐츠 경험을 확장할 수 있습니다.
- 기간: 2026-04-29 - 2026-06-12
- 인원: 5명
- 역할: 큐레이션 플랫폼 서비스의 Backend 개발

## 🔍 구현 기능 상세
### **홍성휘 (소셜/알림 관리)**
<details>
<summary>팔로우 및 글로벌 알림 (Follow & Notification)</summary>

- 팔로우 등록/취소/여부/팔로워 수 조회 엔드포인트 구현
- 자기 자신 팔로우 불가 및 권한 검증 비즈니스 규칙 적용
- 알림 내역 목록 조회 및 커서 기반 페이지네이션 적용
- Kafka를 활용한 비동기 이벤트 기반 아키텍처 도입 (팔로우 요청과 알림 생성 로직 디커플링)
- Kafka Dead Letter Topic (DLT) 및 Recoverer를 적용하여 DB 에러 등으로 인한 알림 생성 실패 시 메시지 격리 및 시스템 마비 방어

</details>

<details>
<summary>1:1 대화방 (Conversation)</summary>

- 대화방 동적 생성/단건 조회/목록 조회 엔드포인트 구현
- 동시 생성 요청 시 Unique 제약조건 충돌 에러를 Catch하여 기존 방을 반환하는 TOCTOU(검사 시간과 사용 시간의 차이) 방어 및 멱등성 보장
- QueryDSL 서브쿼리를 활용하여 상대방 이름 및 최근 대화 내용 동적 키워드 검색 구현
- 첫 페이지에만 totalCount 쿼리를 발생시키는 최적화된 커서 기반 페이지네이션 적용

</details>

<details>
<summary>실시간 채팅 및 접속 상태 관리 (DM & Room Presence)</summary>

- WebSocket 및 STOMP 프로토콜을 활용한 실시간 1:1 메시지 전송 및 과거 내역 조회 구현
- 멀티 인스턴스 환경에 대응하기 위해 Redis Pub/Sub 글로벌 라우팅 아키텍처 구축
- 수위선(Watermark) 기반 메시지 읽음 처리 및 백엔드 측 상태 검증을 통한 중복 API 호출 차단
- 로컬 웹소켓 세션 맵과 Redis 원자적 연산(INCR/DECR)을 결합한 글로벌 접속 상태(Presence) 공유 로직 구현

</details>

<details>
<summary>실시간 이벤트 스트리밍 (SSE)</summary>

- 브라우저 단방향 푸시 알림을 위한 SSE(Server-Sent Events) 커넥션 엔드포인트 구현
- Nginx 및 ALB 등 로드밸런서의 Time-out 연결 끊김 방지를 위한 빈 하트비트(Heartbeat) 스케줄러 구현
- 네트워크 단절 시 알림 유실을 막기 위해 Redis ZSet(Sorted Set)을 활용한 10분간의 이벤트 히스토리 캐싱 적용
- 클라이언트 재접속 시 Last-Event-ID 헤더를 기반으로 ZSet을 탐색하여 놓친 알림을 자동 복구 및 재전송하는 안정성 인프라 구축

</details>

### **김경태 (콘텐츠 관리)**
<details>
<summary>콘텐츠 CRUD</summary>

</details>

<details>

- 키워드 검색(제목/설명/태그) + 단일 정렬 기준 지원
- QueryDSL로 동적 쿼리 구현
- 커서 기반 페이지네이션
- 목록 조회 Redis 캐싱 적용 (TTL 1시간)

</details>

<details>
<summary>콘텐츠 검색 (OpenSearch)</summary>

- nori 기반 한국어 Full Text Search
- bool query로 제목/설명/태그 복합 검색
- Kafka 기반 비동기 인덱싱으로 멀티 인스턴스 환경에서 중복 인덱싱 방지
- fuzziness AUTO로 오타 허용 검색

</details>

<details>
<summary>Spring Batch 콘텐츠 수집</summary>

- TMDB/SportsDB 외부 API 연동 콘텐츠 자동 수집
- ShedLock으로 멀티 인스턴스 환경에서 배치 중복 실행 방지
- 임베딩 생성 실패 시 Micrometer 메트릭 집계

</details>

<details>
<summary>AI 개인화 추천 (pgvector + Gemini)</summary>

- User Embedding = 리뷰한 콘텐츠 embedding DB 평균 (동일 벡터 공간 보장)
- 코사인 거리 임계값 0.3, distance 상위 15건 중 RANDOM() 10건 샘플링
- Redis 캐싱 (TTL 1시간) + 리뷰 변경 시 이벤트 기반 캐시 무효화
- SSE 스트리밍으로 단계별 진행 상황 실시간 전달

</details>

### **황진서 (사용자 관리)**
<details>
<summary>사용자 인증</summary>

- 일반 로그인: /api/auth/sign-in으로 폼 로그인 처리 후 JWT 발급
- OAuth2 소셜 로그인: Google, Kakao OAuth2 로그인 후 JWT 발급
- JWT 기반 Stateless 인증: 요청마다 Access Token을 검증해 SecurityContext에 인증 객체를 설정
- Refresh Token 재발급: Refresh Token은 HttpOnly 쿠키로 관리하고,/api/auth/refresh에서 Access/Refresh Token을 회전
- Redis 기반 토큰 레지스트리: 발급된 토큰의 활성 상태를 Redis에 저장해 로그아웃,
- 계정 잠금, 토큰 회전 시 기존 토큰을 무효화

</details>

<details>
<summary>사용자 권한 변경 & 활동 정지</summary>

- 권한 변경이나 활동 정지 시 로그아웃 구현
- 권한 변경 시 알림, 활동 정지/ 정지 해제시 메일 전송

</details>

<details>
<summary>Remember-me</summary>

- Refresh Token의 만료 시간을 늘리는 방식, Access Token은 그대로 유지, Refresh Token과 Refresh Token 쿠키의 유효기간만 길게 설정

</details>

<details>
<summary>Security</summary>

- @EnableWebSecurity, @EnableMethodSecurity 등 어노테이션으로 웹 보안 활성화
- SecurityFilterChain을 사용해 http 보안 정책 설정
- BCrypt를 사용한 비밀번호의 암호화
- DaoAuthenticationProvider로 이메일/비밀번호 기반 로그인 인증 제공
- 로그인 실패 제한 정책으로 로그인 5회 실패 시 30분간 로그인 잠김
- CORS 설정으로 프론트와 테스트용 로컬만 API 요청을 허용

</details>

<br/>

### **이재준 (웹소켓 관리)**
<details>
<summary>실시간 시청 세션 입장/퇴장</summary>

- /sub/contents/{contentId}/watch 경로 구독 및 해제 이벤트를 추적하여 실시간 인메모리 시청 세션 관리
- 다중 기기 및 멀티탭 환경에서 발생할 수 있는 데이터 경합(Race Condition)을 방지하기 위해 고유 토큰 기반의 Redis 분산 락 및 Lua 스크립트를 활용한 원자적 연산 적용
- 인스턴스 간 물리적 격리를 극복하고 전역 클라이언트에게 세션 변경 정보를 브로드캐스팅하기 위해 Redis Pub/Sub 구조의 전역 메시지 라우팅 아키텍처 도입

</details>

<details>
<summary>실시간 시청 세션 라이브 채팅</summary>

- /sub/contents/{contentId}/chat 경로를 구독하여 STOMP 프로토콜을 활용한 특정 콘텐츠 방 단위의 실시간 메시지 송수신 파이프라인 구축
- 악의적인 패킷 우회를 차단하기 위해 사용자가 해당 콘텐츠의 시청 세션에 실제 참여 중인지 유효성을 상시 검증하는 로직 적용
- 메시지 송수신 처리가 메인 I/O 스레드를 블로킹 되지 않도록 트랜잭션 커밋 시점 기반의 비동기 Redis Pub/Sub 메시지 퍼블리싱 구현

</details>

<details>
<summary>실시간 시청자 목록 조회 시스템</summary>

- 대규모 인원이 밀집한 세션 내에서도 성능 저하 없이 시청자를 스캔할 수 있도록 QueryDSL 기반의 No-Offset 커서 페이징 구현
- 목록 요청 시 마다 데이터베이스에 카운트 쿼리가 중복 호출되는 병목을 방지하기 위해 Redis 전역 분산 카운터 데이터를 캐시로 전면 배치하여 최적화

</details>

<details>
<summary>유저별 실시간 시청 콘텐츠 단건 조회</summary>

- 분산 웹소켓 세션 저장소의 유저 세션 SET을 역추적하여 현재 시청 중인 콘텐츠를 조회
- 다중 세션 키 조회 시 발생하는 네트워크 왕복 비용을 원천적으로 제거하기 위해 executePipelined를 도입하여 단 1번의 연결으로 벌크 명령을 일괄 송신하는 파이프라인 구축
- 활성화된 세션 정보가 존재할 때만 데이터베이스를 1회 조회하도록 구현

</details>

<details>
<summary>실시간 채팅 욕설 감지 및 자동 제재 시스템</summary>

- 부적절한 언어 사용 감지 시 메인 채팅 흐름과 분리된 비동기 이벤트를 트리거하여 경고 누적 및 제재 처리, 알림 생성 로직을 구현
- 누적 경고 횟수에 따라 실시간 채팅을 단계별(3회: 5분, 6회: 1시간, 9회: 서비스 영구 정지)로 차단하는 단계별 제재 비즈니스 규칙 및 UI/UX 비활성화 적용
- 욕설 적발 즉시 해당 유저에게 실시간으로 경고 문구 및 제재 내역을 전달하기 위해 SseService와 연동한 실시간 단방향 푸시 알림 파이프라인 구축
- 폴링이나 상시 스케줄러 가동 없이 Redis Key Expiration 이벤트 리스너를 기반으로 지정된 정지 제한 시간(Redis TTL) 만료 시 유저를 자동으로 정지를 해제하며 실시간 해제 알림을 전송

</details>

<details>
<summary>실시간 채팅 욕설 필터링</summary>

- 문자열 내 다중 패턴을 단 한 번의 스캔으로 찾아내는 Aho-Corasick Trie 자료 구조 기반 고성능 욕설 필터링 시스템
- 욕설 사이에 의도적으로 공백, 숫자, 특수문자를 삽입하여 제재를 우회하는 패턴을 완벽히 차단하도록 구현
- 한글/영문/숫자만 선별하는 정규식 전처리 단계와, 압축된 정규화 문자열의 인덱스와 1:1로 매핑·추적하는 역인덱싱 배열을 구현하여 원본 훼손 없이 정확히 욕설 구간만 가려내는 정밀 마스킹 기술 적용
- 애플리케이션 구동 시점(@PostConstruct)에 파일 시스템으로부터 욕설 사전을 파싱해 자동 인덱싱하는 초기화 로직 구현

</details>

### **선웅제 (콘텐츠 평가 및 큐레이팅 관리)**
<details>
<summary>리뷰(Review) 평가 시스템 및 아키텍처 최적화</summary>

- 도서/콘텐츠에 대한 리뷰 CRUD 엔드포인트 및 비즈니스 로직 구현
- 대규모 데이터 조회 시 발생하는 오프셋 페이징의 성능 저하 문제를 해결하기 위해 No-Offset 방식의 커서 기반 페이지네이션을 적용하여 일관된 조회 성능 확보
- Kafka를 활용한 비동기 이벤트 기반 아키텍처 도입으로 시스템 결합도 완화(Decoupling
- 리뷰 생성 트랜잭션과 평균 별점 및 리뷰 개수 등 후속 통계 업데이트 로직을 물리적으로 분리하여, 핵심 API의 응답 지연 시간을 단축
- 이벤트 발행/구독 모델을 통해 리뷰 등록 트래픽이 몰리는 상황에서도 카프카 메시지 큐가 버퍼 역할을 수행하여, 메인 데이터베이스 및 서버 부하 없이 안정적으로 통계 데이터의 최종 일관성 보장

</details>

<details>
<summary> 플레이리스트(Playlist) 큐레이팅 및 동시성 제어</summary>

- 플레이리스트 생성, 수정, 삭제 및 내부 콘텐츠 추가/삭제, 구독/취소 엔드포인트 구현
- 대용량 플레이리스트 목록 및 구독자 목록 조회를 위한 커서 기반 페이지네이션 적용으로 네트워크 및 DB 조회 성능 최적화
- JPA 비관적 락(Pessimistic Lock)을 이용한 동시성 제어 및 데드락 방지
- 다중 서버 환경에서 여러 사용자가 동시에 동일한 플레이리스트를 구독하거나 콘텐츠를 추가/삭제할 때 발생하는 갱신 손실 및 데이터 정합성 오류 원천 차단
- 트랜잭션 충돌 빈도가 높은 핫스팟(Hotspot) 데이터의 특성을 고려하여, SELECT ... FOR UPDATE 쿼리를 통해 공유 자원에 대한 접근을 직렬화 함으로써 데이터 무결성을 보장하고 시스템 신뢰성을 향상

</details>
